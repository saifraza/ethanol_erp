## The 4 Systemic Problems (Priority Order)

### P0: Multi-Company Data Separation (LEGAL RISK)
**~80 of 154 HIGH issues.** Must fix before MAEL/MGAL go live.

**Root cause (bottom-up):**
1. `shared/config/company.ts` = single hardcoded MSPIL object, no DB lookup
2. `autoJournal.ts` = no companyId param, all journal entries orphaned
3. `eInvoice.ts` / `ewayBill.ts` = hardcoded MSPIL GSTIN in 10+ places
4. Routes = many skip `getCompanyFilter(req)` on GET, skip `getActiveCompanyId(req)` on POST
5. `auth.ts` = `getCompanyFilter` returns `{}` for MSPIL users (sees ALL data)
6. Prisma schema = 4 dispatch models missing `companyId` field entirely; 13 models have orphaned companyId (no @relation)

**Worst files:** accounts.ts (zero filtering), accountsReports.ts (global account lookups), vendorInvoices.ts, vendorPayments.ts, transporterPayments.ts, shipmentDocuments.ts, all inventory stock endpoints

**Fix strategy:**
1. Add `companyId` to 4 missing models (DispatchTruck, DDGSDispatchTruck, SugarDispatchTruck, TransporterPayment)
2. Add `@relation` to Company on all 13 orphaned companyId fields
3. Make `company.ts` DB-backed (lookup by companyId)
4. Add `companyId` param to autoJournal, eInvoice, ewayBill
5. Sweep all routes: GET endpoints need `getCompanyFilter`, POST need `getActiveCompanyId`
6. Factory: add companyId to CachedPurchaseOrder and CachedSupplier

### P1: Raw Error Leaking (SECURITY RISK)
**~25 HIGH issues across ~30 files.**

Pattern: `catch (err: any) { res.status(500).json({ error: err.message }) }` — leaks Prisma schema, column names, constraint errors to client.

**Worst files:** salesOrders.ts, invoices.ts, shipments.ts, freightInquiry.ts, issues.ts, ALL 16 process files

**Fix:** Wrap all handlers in `asyncHandler()` (the shared error handler already sanitizes errors). This is a mechanical find-replace per file.

### P2: Missing Zod Validation on Write Paths
**~50 MED issues.**

Nearly every POST/PUT in procurement, sales, and process modules trusts `req.body` raw. Uses `parseFloat(req.body.field) || 0` instead of Zod schemas.

**Risk:** Malformed input creates garbage records. `|| 0` silently converts NaN to zero (wrong invoice amounts, wrong quantities).

**Fix:** Add Zod schemas to POST/PUT routes. Start with financial writes (invoices, payments, POs).

### P3: Frontend SAP Design Violations (COSMETIC)
**290 rounded violations, 27 shadow-sm, 8 emojis, 13 missing edge-to-edge, 9 missing dark headers across 18 enterprise pages.**

**Worst files:** Shipments.tsx (96 rounded + 8 emojis), entire hr/ directory (6 files, zero compliance), Dashboard.tsx, MaterialMaster.tsx, UsersPage.tsx

**Fix:** Mechanical — search-replace rounded classes, apply SAP tokens from `.claude/skills/sap-design-tokens.md`. Lower priority than data issues.
